### Optimal Size (48x24)

The mobile UI is designed for this size range and displays
all components comfortably with proper spacing.

### Auto-Detection Threshold (60x30)

Terminals up to this size automatically switch to mobile mode
unless `--no-auto-detect` flag is used.

## Color Scheme

- **Warning**: Yellow (back buttons, caution)
- **Error**: Red (errors, destructive actions)
- **Dimmed**: Gray (hints, secondary info)
- **Selected**: Green border + bold text

## Component Patterns

### List with Selection
- Selected item has green border
- j/k to navigate
- Enter to activate

### Cards
- Cyan border for content grouping
- Padding for readability
- Optional title

### Bottom Navigation
- Always visible
- Shows 3-5 tabs
- Active tab highlighted in green
- Keyboard shortcuts [1-5]

### Toast Notifications
- Auto-dismiss after 3 seconds
- Color-coded by type (info/success/warning/error)
- Non-blocking
- Icon + message

### Modal Overlays
- Bold border (yellow)
- "Press Esc to close" hint
- Blocks interaction with main content
- Used for help, actions, confirmations
